## Animation Timeline

```
0.0s   Page load
       └─ Background gradient starts animating
       └─ Particles begin floating

0.3s   Founding Members badge scales in
       └─ Bounce effect

0.5s   Hero headline fades in
       └─ Scale from 0.9 to 1.0

0.7s   Subheadline fades in

0.9s   Countdown timer appears
       └─ Each digit with stagger effect

1.1s   Stats dashboard slides up
       └─ Progress bar fills (2s animation)

1.3s   Purchase widget slides up

1.5s   Benefits grid fades in
       └─ Each card staggers by 0.1s

1.9s   Trust section fades in
       └─ AI agents pop in sequentially

2.2s   Final CTA scales in
       └─ Pulsing effect begins
```

## Glassmorphism Effect Breakdown

```
Traditional Card:
┌────────────────┐
│ ▓▓▓▓▓▓▓▓▓▓▓▓▓ │ ← Solid background
│                │
└────────────────┘

Glassmorphism Card:
┌────────────────┐
│ ░░░░░░░░░░░░░░ │ ← Semi-transparent
│ ░ Blurred bg ░ │   (you see through it)
│ ░░░░░░░░░░░░░░ │
└────────────────┘

CSS Properties:
- background: rgba(15, 23, 42, 0.6)  /* slate-900 at 60% */
- backdrop-filter: blur(24px)         /* blur effect */
- border: 1px solid rgba(6, 182, 212, 0.3)  /* subtle border */
```

## Interaction Flows

### Purchase Flow
```
1. User arrives → Sees countdown + stats
   ↓
2. Scrolls down → Sees purchase widget
   ↓
3. Selects currency → BNB or USDT (button toggles)
   ↓
4. Enters amount → Sees real-time calculation
   ↓
5. Clicks button → Connects MetaMask
   ↓
6. Confirms wallet → Button changes to "Buy Now"
   ↓
7. Clicks Buy Now → Transaction initiates
   ↓
8. Waits for confirmation → Loading state
   ↓
9. Success! → Confetti + success message
```

### Hover Journey
```
1. Mouse over benefit card
   ↓
2. Blur glow appears (0.2s transition)
   ↓
3. Border color changes (slate → cyan)
   ↓
4. Card slightly lifts (subtle transform)
   ↓
5. Mouse leaves
   ↓
6. Everything reverses (0.2s transition)
```

## Testing Scenarios

### Scenario 1: First-time Visitor
```
1. Lands on page → Sees animated hero
2. Reads headline → Understands value prop
3. Sees countdown → Feels urgency
4. Scrolls to stats → Sees social proof (342 members)
5. Sees progress bar → Understands scarcity
6. Reaches purchase → Ready to buy
```

### Scenario 2: Mobile User
```
1. Opens on phone → Loads quickly (<2s)
2. Scrolls smoothly → No lag on animations
3. Taps BNB button → Switches to BNB
4. Types amount → Keyboard doesn't overlap input
```

```
2. Scrolls to stats → Raised amount increased
3. Sees member count → More people joined
```
